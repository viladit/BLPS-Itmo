# Проверка приложения

## 1. Поднять SSH-туннель до учебного сервера

Если локальный `5432` свободен:

```bash
ssh -p 2222 -L 5432:localhost:5432 s368925@helios.cs.ifmo.ru
```

Если локальный `5432` занят:

```bash
ssh -p 2222 -L 15432:localhost:5432 s368925@helios.cs.ifmo.ru
```

Тогда перед запуском приложения нужно задать:

```bash
export DB_PORT=15432
```

## 2. Запустить приложение

```bash
mvn spring-boot:run
```

Что важно:

- приложение слушает `http://localhost:8080`
- таблицы в PostgreSQL создаются автоматически Hibernate при старте
- если база доступна, в логах не будет ошибок подключения к PostgreSQL

## 3. Проверить, что API отвечает

```bash
curl http://localhost:8080/api/orders
```

Ожидается JSON-массив, например `[]`, если заказов пока нет.

## 4. Проверить полный бизнес-процесс

Из корня проекта:

```bash
chmod +x scripts/curl/*.sh
./scripts/curl/run_happy_path.sh
```

Скрипт:

- создаёт заказ
- проверяет остатки
- резервирует товары
- подтверждает заказ
- создаёт picking task
- помечает заказ как собранный
- упаковывает заказ
- передаёт в доставку
- завершает заказ

## 5. Проверить сценарий отмены

```bash
./scripts/curl/create_order.sh
./scripts/curl/cancel_order.sh 1
```

Вместо `1` подставьте реальный `orderId`.

## 6. Проверить отдельные эндпоинты вручную

Примеры:

```bash
./scripts/curl/list_orders.sh
./scripts/curl/get_order.sh 1
./scripts/curl/check_stock.sh 1
./scripts/curl/reserve_items.sh 1
./scripts/curl/handoff_to_delivery.sh 1
```

## 7. Проверить тесты

```bash
mvn test
```

## 8. Что смотреть при защите

- что туннель поднят и приложение стартует без ошибок
- что таблицы создаются автоматически в базе `studs`
- что happy path проходит по всем статусам
- что невалидные переходы возвращают ошибку `400`
- что несуществующий заказ возвращает `404`
